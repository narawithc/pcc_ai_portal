## Precise Corporation Group — ฉบับ v1.0

**สถานะ:** ฉบับร่าง | **มีผลบังคับใช้:** ___/___/2026

---

## 1. วัตถุประสงค์

นโยบายฉบับนี้กำหนดแนวทางการใช้ปัญญาประดิษฐ์ (AI) อย่างปลอดภัย มีจริยธรรม และเกิดประโยชน์สูงสุด สำหรับพนักงานทุกคนในกลุ่มบริษัท Precise Corporation

---

## 2. ขอบเขต

นโยบายนี้ใช้บังคับกับ:
- พนักงานทุกระดับ ทุกบริษัทในกลุ่ม
- การใช้ AI ทุกรูปแบบ ทั้งที่องค์กรจัดหาและเครื่องมือส่วนตัวที่ใช้ในงาน
- ครอบคลุม AI Gateway (`ai.precise.co.th`), Microsoft Copilot และ AI อื่นที่ใช้ในงาน

---

## 3. หลักการ 5 ข้อ

| # | หลักการ | สาระสำคัญ |
|---|---------|----------|
| 1 | **ข้อมูลต้องปลอดภัย** | ห้ามป้อนข้อมูลลับเข้า AI สาธารณะ ต้องจำแนกระดับข้อมูลก่อนใช้ทุกครั้ง |
| 2 | **มนุษย์ตัดสินใจ** | AI เป็นผู้ช่วย ไม่ใช่ผู้ตัดสินใจ ต้องตรวจสอบผลลัพธ์ก่อนใช้งานเสมอ |
| 3 | **โปร่งใส** | งานที่ใช้ AI ช่วยต้องระบุ "AI-Assisted" เมื่อเหมาะสม |
| 4 | **เป็นธรรม** | ห้ามใช้ AI เลือกปฏิบัติ ต้องตรวจสอบ bias ในผลลัพธ์ |
| 5 | **ปฏิบัติตามกฎหมาย** | ปฏิบัติตาม PDPA และกฎหมายที่เกี่ยวข้อง |

## 4. การจำแนกข้อมูล 4 ระดับ

> **ระบบตรวจจับอัตโนมัติ:** AI Gateway วิเคราะห์ prompt ก่อนส่งไป LLM — ถ้าพบสัญญาณข้อมูลลับ ระบบจะแจ้งเตือนหรือบล็อกอัตโนมัติ

| ระดับ | คำอธิบาย | ใช้กับ AI ได้? | ตัวอย่าง |
|-------|---------|--------------|---------|
| 🟢 **Public** | ข้อมูลเผยแพร่สาธารณะ | ✅ ใช้ได้ทุกเครื่องมือ | ข่าวสาร, FAQ, ข้อมูลผลิตภัณฑ์ทั่วไป |
| 🔵 **Internal** | ข้อมูลภายในทั่วไป | ✅ เฉพาะ Enterprise tools (Portal/Copilot) | SOP, รายงานภายใน, meeting notes |
| 🟡 **Confidential** | ข้อมูลลับ มีผลกระทบหากรั่วไหล | ⚠️ Enterprise API ที่มี DPA เท่านั้น + ต้อง review | สัญญา, ข้อมูลลูกค้า, ตัวเลขการเงิน |
| 🔴 **Top Secret** | ข้อมูลลับสุด | ⛔ ห้ามใช้กับ AI ภายนอก (ใช้ Local Ollama เท่านั้น) | M&A targets, salary list, source code proprietary |

### สัญญาณที่ระบบตรวจจับ (Auto-detection)

**Top Secret triggers (บล็อกอัตโนมัติ):**
- คำว่า "ลับสุด", "top secret", "M&A", "acquisition", "takeover"
- "salary list", "payroll", "compensation structure"
- API keys, passwords, secrets ในรูปแบบ regex
- Source code ที่มี proprietary markers

**Confidential triggers (แจ้งเตือน + require review):**
- ข้อมูลส่วนบุคคล (เลขบัตรประชาชน, เบอร์โทร, อีเมล, เลขบัญชี)
- ชื่อลูกค้า + จำนวนเงิน
- หมายเลขสัญญา/PO
- ตัวเลขการเงิน + ช่วงเวลา (quarterly, annual)

## 5. เครื่องมือ AI ที่อนุมัติ

### 5.1 AI Portal — ai.precise.co.th (LiteLLM Gateway)
- Login ด้วย email องค์กร (`@precise.co.th`) ผ่าน Azure AD SSO
- Model ที่ใช้ได้ตาม Tier ที่ได้รับ (Standard / Professional / Premium)
- ทุก request ถูกบันทึก (audit log)

### 5.2 Microsoft Copilot
- ใช้งานผ่าน M365 ที่องค์กรจัดหาให้
- ข้อมูลอยู่ใน M365 Tenant

### 5.3 เครื่องมือที่ห้ามใช้ในงาน
- AI ที่ไม่มี Enterprise license / ไม่มี data protection agreement
- Deepfake tools ทุกกรณี
- AI ที่ deploy เองโดยไม่ผ่านอนุมัติจาก PDE
- Public ChatGPT (ไม่มี DPA กับ Precise)
- Public Gemini (ไม่มี DPA กับ Precise)

> รายการเครื่องมือที่อนุมัติ/บล็อกฉบับเต็มอยู่ใน AI Gateway → `/api/v1/admin/tools`

## 6. สิ่งที่ต้องทำ (Do's)

- ✅ ใช้บัญชี Enterprise ที่องค์กรจัดหาให้
- ✅ ตรวจสอบระดับข้อมูลก่อนป้อนเข้า AI ทุกครั้ง
- ✅ ตรวจสอบความถูกต้องของผลลัพธ์จาก AI ก่อนนำไปใช้
- ✅ ลบ/ปกปิดข้อมูลส่วนบุคคล (PII) ก่อนส่งเข้า AI
- ✅ รายงานเหตุการณ์ผิดปกติทันทีผ่าน `ai-report@precise.co.th` หรือ AI Portal → Report Incident
- ✅ เข้าร่วม AI Training Program ตามที่กำหนด

---

## 7. สิ่งที่ห้ามทำ (Don'ts)

- ❌ ป้อนข้อมูล Top Secret เข้า AI ทุกประเภท (ยกเว้น Local Ollama)
- ❌ ป้อนข้อมูลส่วนบุคคล (เลขบัตรประชาชน, เลขบัญชี, เบอร์โทร) โดยไม่ปกปิด
- ❌ ใช้ AI สร้างเนื้อหาหลอกลวง, deepfake หรือข้อมูลเท็จ
- ❌ นำผลลัพธ์ AI ไปใช้โดยไม่ตรวจสอบ ในงานที่มีผลทางกฎหมาย/การเงิน
- ❌ ส่ง source code ที่เป็น proprietary เข้า AI สาธารณะ
- ❌ แชร์บัญชี AI หรือ API keys ให้ผู้อื่น
- ❌ ใช้เครื่องมือ AI ที่ไม่ได้รับอนุมัติในการทำงาน

---

## 8. ทรัพย์สินทางปัญญา

- ผลงานที่สร้างโดย AI ด้วยทรัพยากรขององค์กร → เป็นทรัพย์สินขององค์กร
- ต้องตรวจสอบว่าผลลัพธ์จาก AI ไม่ละเมิดลิขสิทธิ์ผู้อื่น
- Code ที่สร้างโดย AI ต้องผ่าน code review ก่อน deploy

## 10. การรายงานเหตุการณ์

เมื่อพบเหตุการณ์ผิดปกติ (ข้อมูลรั่ว, AI ให้ผลลัพธ์ผิดพลาดร้ายแรง, ถูก misuse):

1. **แจ้งทันที** → `ai-report@precise.co.th` หรือ AI Portal → **Report Incident** (`/incidents/new`)
2. **PDE + CISO ประเมินและหยุดยั้งภายใน 4 ชั่วโมง**
3. **สอบสวนภายใน 24 ชั่วโมง**
4. **แก้ไข + บันทึกเข้า KM + อัปเดตนโยบาย**
5. ถ้าเป็น data breach → **แจ้ง PDPC ภายใน 72 ชม.** ตาม PDPA
6. ผู้รายงานได้รับการคุ้มครองตามนโยบาย Whistleblower

> รายละเอียด runbook อยู่ใน `docs/incident-runbook.md`

### ประเภทเหตุการณ์ (สำหรับกรอกรายงาน)

| category | ตัวอย่าง |
|----------|---------|
| `data_leak` | ส่งข้อมูลลับออกนอกองค์กรโดยไม่ตั้งใจ |
| `hallucination` | AI ให้ข้อมูลผิด นำไปตัดสินใจทางการเงิน/กฎหมาย |
| `misuse` | ใช้ AI สร้าง deepfake, spam, harassment |
| `deepfake` | พบเนื้อหา deepfake ที่เกี่ยวกับพนักงาน/ผู้บริหาร |
| `other` | เหตุการณ์อื่นที่ต้องรายงาน |
